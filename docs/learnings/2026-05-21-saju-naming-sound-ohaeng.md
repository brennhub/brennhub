## 1. 정찰 — 음령오행 작명 이론 (팩트 기반, 출처 명시)

음령오행(音靈五行) = 소리오행 = 발음오행. 한글 자음의 오행. 작명에서 성+이름
자음의 오행 배열이 **상생이면 길, 상극이면 흉**.

### 자음 오행 배속
- 牙音 ㄱㅋ → 木 / 舌音 ㄴㄷㄹㅌ → 火 / 齒音 ㅅㅈㅊ → 金 — **3종 비논쟁**.
- 脣音 ㅁㅂㅍ · 喉音 ㅇㅎ → **학파 분기**:

| 학파 | 脣音(ㅁㅂㅍ) | 喉音(ㅇㅎ) | 근거 |
|---|---|---|---|
| 훈민정음 해례 제자해 원전 (1446) | 土 (宮聲) | 水 (羽聲) | 훈민정음 해례 |
| 신경준 『훈민정음운해』 (1750) 이후 | 水 | 土 | 신경준 — 원전 미열람 추정 |
| **현대 작명 실무 통용 다수안** | **水** | **土** | 신경준 계열 계승 |

신경준이 훈민정음 원전을 보지 못한 채 순음·후음 배속을 뒤바꿨고, 후대 작명
실무가 이를 다수 계승. 두 안 모두 실재 — 우열 표기 없음 (brennhub 원칙).

### 채점법 — 상생/상극 배열
- 상생: 木生火 火生土 土生金 金生水 水生木 → 길.
- 상극: 木克土 土克水 水克火 火克金 金克木 → 흉.
- 적용 위치 **3학설**: A(초성만) / B(초성·종성 포함) / C(A·B 둘 다).

### 출처
- 김만태, 「훈민정음의 제자원리와 역학사상」, 『철학사상』 2012 (SNU s-space).
- 김만태, 「자음에서 순음과 후음의 오행 분별」 (한국명리성명학회).
- miso.co.kr 「소리오행」 / sajuforum.com 「발음오행의 상생」 — 실무 채점법·3학설.
- 1차 사료: 훈민정음 해례 제자해(1446) / 신경준 『훈민정음운해』(1750).

## 2. 현재 코드의 문제 (lib/names.ts)

- `calcSoundScore` — 음령오행을 표방하나 실제론 "각 글자 초성오행 ∈ yongsin → +50"
  멤버십 체크. **상생/상극 배열 이론 전혀 미반영.**
- `calcOhaengScore` — `c.ohaeng`(음령오행 컬럼) 채점 = `calcSoundScore`와 **동일 축
  이중 채점** (F3). 자원오행 `ja_ohaeng`은 점수에 없음.

## 3. 설계 — sound-ohaeng.ts (C-5-4 radical-ohaeng.ts plug-in 패턴 복제)

- **학파 plug-in**: `SoundSchool = "tongyong" | "hunminjeongeum"`.
  `CONSONANT_OHAENG[school]` — 脣音·喉音만 분기, 나머지 공통. default `tongyong`.
- **ruleset plug-in**: `SoundRuleset = "choseong" | "with-jongseong" | "both"`
  (A/B/C 학설). default `choseong`.
- **상생/상극**: `OHAENG_GENERATES`/`OHAENG_CONTROLS` + `relateOhaeng(a,b)` →
  상생/비화/상극.
- **채점**: 오행 체인의 인접쌍 → `RELATION_POINT` (상생 1.0 / 비화 0.5 / 상극 0.0)
  평균 ×100. 상생/상극 길흉은 **이론 팩트**, 점수값은 **캘리브레이션**(39-C).
- **받침 브릿지** (B/C 핵심): 초성·종성을 음절 순서대로 한 체인으로 펼침
  `[초성1, 종성1?, 초성2, 종성2?, …]`. 이름 초성끼리 상극이어도 앞 글자 종성이
  중간 오행이면 자연히 상생 연쇄가 됨.
  - 검증 예: `삼구` — 초성 [금,목] 금극목 → A학설 0점. 종성 ㅁ(수) 삽입 →
    [금,수,목] 금생수·수생목 → B학설 100점. (PoC case 4)
- **겹받침**: 표준발음법 대표음으로 발음 자음 환원 (ㄳ→ㄱ, ㄻ→ㅁ 등) — 작명
  학파 주장이 아닌 한국어 표준발음 사실 기준.

## 5. 상생 방향성 정찰 (2단계 진입 전)

음령오행 상생 채점이 **방향 무관 관계**인지(목·수 인접 = 수생목 관계 → 상생),
아니면 발음 순서 방향성(정방향 木生火 순서만 길)을 보는지 추가 정찰.

- **다수안 = 방향 무관.** 주요 이론 페이지(miso.co.kr 소리오행 등)가 상생을
  *관계*로 제시 — 변주 축은 **위치**(초성/종성 = A·B·C 3학설)이지 방향이 아님.
  "정방향만 길 / 역방향 흉" 학설은 정찰에서 미발견.
- 방향성 refinement 이론("이름 끝글자→성씨 방향 상생이 최선, 성씨→이름은 차선")이
  1회 언급됐으나 verbatim 1차 출처 핀포인트 실패(sajuforum TLS 차단). 게다가 그조차
  **양방향 모두 '상생(길)'**으로 보는 우열 순위 — 상생 vs 흉 구분이 아님.
- **채택**: `relateOhaeng` 방향 무관 유지 (다수안). 방향성을 학파 plug-in으로
  분기할 만큼 확립된 대립 학설은 미확인 — brennhub "추측 금지" 원칙상 보류.
- **향후**: 1차 출처 확보 시 `directionBonus`(상생 내 방향 가산점) refinement
  plug-in으로 추가 가능 — 학파 분기가 아닌 선택적 가산. BACKLOG 별도 task 등록.
