# 태그잇 (Tag-it) 추출 규칙 — 근거·계측·판단 (RULES)

> **이 문서의 목적.** tag-it 키워드 추출의 모든 규칙 결정을 "현재 처리 / 근거·계측 / 한계 /
> 재검토"로 한 곳에 쌓는다. `weights.ts`엔 숫자만 남고 "**왜 이렇게 했는가**"가 흩어져 있어,
> 같은 질문("흘리가 명사야?", "원의는 왜 안 깎여?")이 반복됐다 — 밑빠진 독. 이 문서가 그 토대다.
>
> **운영 규칙.** 추출 규칙을 바꿀 땐 **이 문서를 먼저 갱신**하고 코드는 그다음. 모든 채택·보류
> 결정은 충돌 수 등 실측 근거와 함께 여기 기록한다. (BRENNHUB 정신: "사전은 규칙을 버릴
> 대상이 아니라 규칙을 발견·검증하는 토대." "가중치는 우리 데이터 안의 잠정 추정이지 언어의
> 진실이 아니다 — 보수적으로." "세상은 흑백이 아니다 — 확정 못 하는 건 확정 안 한다.")
>
> 관련: 코드 `lib/tag-it/{extract,weights,tokenize,stopwords.ko}.ts` · 이력 `../../app/tools/tag-it/CHANGELOG.md` · 향후 작업 `../../app/tools/tag-it/BACKLOG.md`.

## 0. 측정 방법 (재현용)

- **사전**: `public/tag-it/noun-dict.txt.gz` = mecab-ko-dic 2.1.1 NNG(일반명사) 한글 음절만,
  **216,932개**. `decodeNounDict()`로 Set 로드.
- **충돌(collision)**: "그 어미(또는 조사)로 끝나는 사전 명사 수". 모델의 매칭 규칙과 동일하게
  `noun.length > pattern.length && noun.endsWith(pattern)`로 센다. (어미 매칭만 0.8.1부터 `>=`.)
  - 충돌 ↓ = "명사가 아니라 용언 활용"이라는 신호가 확실 → 큰 차감 안전.
  - 충돌 ↑ = 그 글자는 명사 말음 → 큰 차감 시 명사 학살.
- 모든 수치는 **실측**(추측 금지). 측정 안 한 항목은 "미측정"으로 명기.

## 1. 명사 보호 사전 (NNG)

- **현재**: NNG 216,932개를 정적 gzip(0.75MB)으로 1회 lazy-load → 확률 모델의 **보호 신호**.
  사전 O = 100% 시작, 사전 X = 50% 시작. fetch/디코드 실패 시 fallback(사전 없이 50% 경로).
- **근거**: 무한한 한국어 활용을 blacklist로 못 따라감 → "무엇이 명사인가"의 positive 신호로 전환.
  한자(㢿 등) 제외·한글 음절만(키워드 추출엔 한글). NNP(고유명사) 제외(희소 복합, 0.8.0 결정).
- **한계**: 사전에 **조사 융합 표제어**(원의/원이/돈은)와 **동음이의**(하지=夏至)가 섞여 있음 → §5,§13.
- **재검토**: SCHEMA_VERSION 성격의 사전 갱신 시 재생성 스크립트(`scripts/gen-noun-dict.mjs`).

## 2. 명사 확률 모델 (핵심)

- **현재** (`extract.ts §nounProbability`, 상수 `weights.ts`):
  - 사전 O: `p = 100 − min(decr, 15)` → **하한 85**(floor, 추락 봉쇄).
  - 사전 X: `p = 50 − min(decr, 50)` → 하한 0.
  - `decr = 어미(longest-match 1개) + 신호1(용언 패러다임? 30) + 기능어간(있/없/같…? 60)`.
  - **강도 = 채택 임계** `THRESHOLD {1:0, 2:0, 3:20, 4:30, 5:40}`. `p >= 임계`면 채택.
- **근거**: floor 85 ≥ 임계 40 → **사전 명사는 확률 모델로 절대 제거 안 됨**(구조적 보장).
  비사전 명사(차감 0)는 50에 모이는데 임계 최대 40 < 50이라 전 강도 보존, 비사전 노이즈(차감 후
  ≤15)만 강도 3+에서 제거 — "50점 경계"를 임계 40으로 분리(0.8.0).
- **⚠️ 결함(재검토 §A-1)**: `decr`가 사실상 **이분법**(어미가 weights에 있으면 차감, 없으면 0).
  "흑백 아닌 확률"을 표방하면서 위험 어미를 0/45로 흑백 처리 중. 위험 어미는 0이 아니라
  **아주 작은 가중치(1~3)**를 줘야 의심을 표현하면서 명사를 안 죽인다. → §A-1.

> **긴 형태는 안전**: `~하려=0 · ~려고=0 · ~더라=0 · ~하려고=0 · ~더니=1 · ~해도=7`.
> → "시작하려/하려고/~더라" 류는 longest-match로 안전하게 추가 가능(미적용 = 측정 안 한 영역).

## 5. 1글자 조사 (의/이/은/을/가/도/로/과/에) — w0

- **현재**: 어미 가중치 **0**(차감 안 함). 따라서 `원의`(사전O)=100%, `돈은`=96%(은 w4만).
- **근거(계측)**: 이 조사로 끝나는 **사전 명사가 대량** — ~의 1430·~이 6403·~가 1367·~도 1487·
  ~과 1238·~로 681·~은 203·~에 204. 큰 차감 시 **회의/차이/평가/강도/결과/도로** 학살.
  알려진 정상명사 59개 중 **55개**가 "어간∈사전"이라 융합 오판(원의=원+의 와 회의=會議 구분 불가).
- **한계**: `원의/원이/돈은`을 못 깎음(사전O + 조사 w0). 표면 정보로 융합/정상 구분 불가.
- **⚠️ 재검토 §A-1**: w0(완전 무시)이 아니라 **w1~3(작은 차감)**이 맞다. 원의→99%(살짝 의심),
  회의(사전O)→99%(거의 영향 0, 안전). 작은 차감은 명사를 안 죽이며 의심만 표현 — floor·확률
  철학의 진짜 구현. 현재 0은 신호를 죽인 이분법 결함.

## 6. 긴 조사 과도제거 완화 (e) — `KO_JOSA_OVERSTRIP`

- **현재**(0.8.2): `stripJosa`는 보통 떼고 남는 어간 1음절이면 보존(회의→회 방지)하나, **긴(2글자+)
  격·보조사 16개**(에서/으로/부터/에게/한테/로부터/에게서/한테서/으로서/으로써/에서부터/
  으로부터/처럼/보다/밖에/커녕)에 한해 1음절 어간도 분리. 등에서→등, 금액으로→금액.
- **근거**: 이 조사들로 끝나는 사전 명사 ≈0(에서=1·으로=1 노이즈 외) — 1글자 조사와 차원이 다름.
- **제외**: 까지(7)/만큼(6)/라도(9)/조차(4)/마저(1)/마다(2) — 소수라도 충돌 있어 보존(등까지→등까지).

## 7. 단독 조사 토큰 제외 (refine)

- **현재**(0.8.3): refine에서 토큰이 `KO_JOSA` 멤버면 제거. `금액(으로)`·`회사 에서`처럼
  구두점·띄어쓰기로 조사가 독립 토큰화된 것(으로/에서/에게서)을 떨군다.
- **근거**: 순수 조사는 키워드 아님. stripJosa가 조사를 배출하는 게 아니라 **처음부터 조사만인
  토큰** 제거. 2글자+ 조사 중 사전 등재 7개(에서/까지/조차…)도 키워드 가치 0.

## 8. 명사 어간 복원 (denominal, `extractNounStem`)

- **현재**(강도 3+): "명사+하다/되다/시키다/당하다+어미"에서 접미 용언을 떼어 명사 복원
  (분석했다→분석, 평가되었다→평가). 과분리 가드: 남는 어간 한글 ≥2.
- **근거**: 접미 사전이 하/되/시키/당하 계열뿐이라 일반 동사(먹었다)는 매치 안 됨.
- **한계**: 조사 strip보다 먼저 실행(포함하는→포함). 복원된 토큰은 차감 면제(이미 명사 어간).

## 9. 신호1 — 용언 활용 패러다임

- **현재**: 토큰의 어간(검증 어미 `KO_INFL_ENDINGS`만 떼어)이 **문서 안에서** 용언 활용
  (`KO_VERBAL_MARKERS`: 하다/되다/었다…)과 함께 나타나면 용언으로 보고 차감(w30). 기능어간
  (있/없/같…)은 빈도 무관 차감(w60), borderline(보=報告)은 빈출이면 보존(빈도 가드 5).
- **근거**: 임의 절단 금지(보고서→보 학살 방지) — `KO_INFL_ENDINGS`(진짜 어미)만.
- **⚠️ 한계 §A-2**: 어간이 `KO_INFL_ENDINGS`로 안 끝나면 못 잡음. `흘리`는 "리"로 끝나
  inflectedStem=null → 신호1 미발동. → §A-2.

## 10. 형태 노이즈 필터 (강도별, 토큰 형태 기준)

- 기호·비문자(전 강도) / 숫자+영문 단편(강도2+, 5px·F1) / 숫자+세는단위(강도3+, 2가지·3번) /
  URL(강도5) / 1음절 의존명사+조사(강도5, 것은·등이). 시간·물리 단위(30분·5km)는 보존.

## 11. 영문 토큰 — 기본 포함 (0.8.4, §C 해결)

- **현재**: 영문은 **한글과 동일 취급 — 기본 포함, 길이 기준 제거·감점 없음**. 영어 불용어
  (`stopwords.en`)·기호·숫자단편(`isLatinDigitFragment`: 5px/F1)만 거른다. KPI/ROI/API/AI/UX/
  EBTS/Accel 전부 전 강도 생존.
- **근거(왜 감점도 없앴나 — A-1 철학과 일관)**: 한글은 NNG 사전이라는 **판별 근거**가 있어 확률을
  매기지만, 영문은 사전이 없어 KPI(키워드)와 Math(노이즈)를 구분할 데이터가 없다. 영문에 감점을
  주면 그건 "불확실성 표현"이 아니라 **근거 없이 KPI/EBTS까지 깎는 것** → A-1("근거 있는 차감")과
  어긋남. 그래서 영문은 깎지 않고 포함, **일반 영단어 노이즈(Math/Letter)는 빈도 long tail로
  접히거나 사용자가 칩에서 거른다**("근거 없이 깎지 않는다 + 못 거르는 건 사용자/노출로").
- **이전 결함(해결됨)**: 0.8.3까지 `isShortAbbrev`(한글X·len≤3·숫자X)가 강도 3·4 점수 감점 +
  **강도5 제거** → KPI/ROI/API/AI/UX 학살(EBTS len4·Accel len5는 생존). 길이 기준 제거·감점은
  근거 없는 오류라 0.8.4에서 `dropShortLatin`·`abbrevPenalty`·`isShortAbbrev` 전부 삭제.

## 12. 강도 프로파일 (1~5)

- 1 관대(기호만) · 2(+숫자영문단편) · 3 균형(+조사·명사복원·카운터, 임계20) · 4(+약어감점, 임계30)
  · 5 엄격(+짧은영문제거·URL·1음절, 임계40). 확률(hover %)은 강도 무관 동일, **채택만 임계가 가름**.

## ★ 미해결 · 재검토 항목 (다음 task 후보)

> 아래는 **이 문서 작성 중 계측으로 확인된 설계 결함/한계**다. 이번(문서화) task에선 **수정하지
> 않고 기록만** 한다. 수정은 별도 task에서 이 근거를 바탕으로.

### A-1. 위험 어미 0 차감 = 이분법 결함 (우선순위 높음)
- **증상**: 원의=100·돈은=96. 1글자 조사·위험 어미가 w0이라 의심조차 표현 못 함.
- **원인**: "보수적"을 "큰 차감 금지"가 아니라 "차감 0"으로 구현 — 흑백.
- **방향**: 위험 어미/1글자 조사에 **w1~3 작은 차감** 부여. 사전O는 floor라 거의 영향 0(99%),
  비사전 노이즈는 살짝 낮아짐. 명사 안 죽이며 의심 표현. **계측**: 작은 차감이 명사 30/30·
  사전명사 회귀에 영향 없는지 확인 후.

### A-2. 잘못된 분석 단위 — 흘리 / 흘리는 (우선순위 높음)
- **증상**: 본문 "흘리는"인데 후보는 "흘리"(50%). `splitTokens("흘리는")=흘리는` →
  `stripJosa("흘리는")=흘리`(관형형 어미 "는"을 조사 "는"으로 오인해 제거) → 동사 어간 "흘리"만 남음.
- **원인**: ① stripJosa가 용언 관형형 "는/은/을"을 조사로 떼어 **동사 어간을 노출**.
  ② 신호1은 "흘리"가 "리"로 끝나 inflectedStem=null → 못 잡음. ③ "리"는 충돌 5656이라 어미
  규칙으로도 못 잡음. → 세 겹으로 빠져나감.
- **방향**: stripJosa 전/후의 **원형도 함께 검사**(흘리는을 통째로 어미 판정), 또는 "는/은/을" strip
  결과가 비명사 어간일 때 원형 어미(는 w45)로 차감. 단위 판별을 stripJosa에 뺏기지 말 것.
- **관련**: `물을`(46)도 같은 류(물+을, 가드가 통째 유지 + 1글자 조사) — §5/§A-1과 연결.

### B. 측정 안 한 어미 — 못한/나눠주면/인정하더/시작하려
- **증상**: 50% 잔존(차감 0). 해당 어미가 weights에 없음.
- **계측(이번 실측)**: 1글자 ~한(327)/~면(524)/~더(86)/~려(173)는 高충돌(위험). **그러나 긴 형태
  ~하려(0)·려고(0)·더라(0)·하려고(0)·더니(1)·해도(7)는 안전.**
- **방향**: 긴 안전 어미(하려/려고/더라/하려고…) **추가**. 1글자 위험 어미는 §A-1의 작은 차감으로.
  "직무유기"가 아니라 "측정 안 한 영역" — 측정했으니 일부 채택 가능.

### C. 영문 ≤3글자 약어 강도5 제거 = 설계 오류 — ✅ 해결 (0.8.4)
- **증상(실측)**: 강도5에서 KPI/ROI/API/AI/UX 제거(EBTS/Accel/Math/Letter는 생존).
- **해결**: `dropShortLatin`(강도5 제거) + `abbrevPenalty`(감점) + `isShortAbbrev` **전부 삭제** →
  영문 기본 포함(§11). 감점까지 없앤 이유: 영문은 사전(판별 근거)이 없어 감점이 "근거 없는 차감"
  (KPI/EBTS까지) = A-1 철학 위반. 일반 영단어 노이즈는 long tail/사용자 거름. `isLatinDigitFragment`
  (5px/F1 숫자단편)는 영문 약어와 무관한 진짜 노이즈라 유지.

### D. 사전 오염 — 원의/원이/돈은 (별도 task, 보류 유지)
- 1글자 조사 한계(§5) + 사전 등재. 표면+사전조회로 융합 vs 정상(原義 vs 원+의) **판별 불가**.
- **방향(보류)**: (d) 문서 빈도 기반 병합 — 어간이 같은 문서에 빈출이고 `freq(어간)≫freq(융합형)`
  이면 융합 처리. **차이/강도/회의 오제거 위험** → 반드시 실EBTS 계측 + 보수 파라미터(≥3배) 후만.
- **현재 결정**: 그냥 둔다(저빈도 잡음 < 오제거 비용). "原義인지 원+의인지 문맥 없이 모른다 →
  함부로 안 죽이고 100%로 둔다" = 확정 못 하는 건 확정 안 함. 완전 판별 = 문맥/의미 = **AI 영역**.

### E. 동음이의 — 하지(夏至/下肢 vs 하다+지)
