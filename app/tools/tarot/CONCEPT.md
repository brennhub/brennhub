## 0. 한 줄 정의 `[합의]`

> **시장 전체가 "AI가 매번 지어내는 신탁"이라면, 우리는 "변하지 않는 사전을 정직하게 보여주는 타로"다.**
> 결정론 카드 사전(신뢰축) + 비효율적 의식·정/역 선택(몰입축)으로 승부한다.

차별 한 문장: **"우리는 지어내지 않는다."**

미래 예측에 대한 입장: 단정하지 않는다. 미래를 보든 마음을 보든 의미를 완성하는 것은 사용자다. 서비스는 카드의 고정 의미를 정직하게 전달하고 몰입을 설계할 뿐. ("미래를 맞히지 않는다"류 문구를 대외 슬로건으로 쓰지 않는다.)

## 1. 시장 분석 — 왜 빈틈인가

**Tarotap** (tarotap.com): 풀-AI 챗봇. 페르소나 리더 6명(같은 카드도 누가 읽느냐로 해석 변화) + 질문→스프레드 자동추천 + 리딩 후 대화 연속 + 퀵/딥 2모드. 20개 언어, VIP 구독.

**Jenova** (jenova.ai): 범용 AI 에이전트 플랫폼의 타로 에이전트. 78장 RWS 완전 + 시각적 스프레드 이미지 생성(양초·수정 분위기) + 단일 대화 내 연속성 + 친숙도별 해석 깊이. 무료티어 + 구독.

**공통 약점 = 우리 빈틈:**
1. 둘 다 풀-AI → 매 리딩 LLM 호출 (비용 누적 + "AI가 다 해줌")
2. 둘 다 "정확함"을 판다 — 본질적으로 검증 불가능한 주장
3. 비주얼 평범/생성형 — 고유 아이덴티티 없음
4. 구독 수익화 — brennhub 12개월 금지와 충돌
5. 공유·바이럴 장치 거의 없음 (혼자 보고 끝)
6. "연속성"이 단일 대화 안으로 제한 — 장기 저널 약함

**스트레스 테스트 결과 (대안 기각 이력)**: AI-first(비용+동질화로 기각) / 데일리 1장 first(차별점 미노출로 기각) / "타로" 간판 제거·성찰카드 리브랜딩(검색량·문화 카테고리 자산 포기라 기각). → 현 방향 유지.

## 2. 포지셔닝 & brennhub 필터 통과 `[합의]`

- **신뢰축**: 결정론 사전 = 환각 없음. "정답 생성"이 아니라 "고정 의미의 정직한 전달"이므로 **AI가 필수가 아니다.**
- **몰입축(효능감)**: 의식·정/역 선택이 만드는 진지함 = 실사용 효용. 시각적 완성도가 아니라 몰입.
- **거울 인사이트 (내부 설계 원리)**: 의미는 사용자가 스스로 완성한다(바넘 효과 + 사용자의 고교 시절 90% 실증 — 해설집 건조 인용만으로 공감 발생). → 건조한 키워드 인용으로 충분, 현란한 해석 불필요. 단, 이는 설계 원리이지 대외 포지션이 아님.
- **외부 기획서 필터 적용 결과**: 가격/결제 제외 · 광고 제외 · 별도 백엔드 없음(Workers+D1 유지) · 수익화 12개월 금지 → **AI 비용 회피가 원칙과 정합**.
- **포트폴리오 시너지**: 사주작명 + 타로 = **결정론 운세 클러스터** 형성. 후속 후보(결정론 데일리 운세)까지 가면 상호 트래픽 자산.

## 3. AI 정책 — 현재 결정 = (c) `[합의]`

- **1차 출시: 결정론 100%.** AI 호출 0, 완전 client-side, 비용 0, 6개월 후에도 손 안 감.
- **AI는 기본 구축 완료 후 별도 검토.** 붙이더라도 핵심 쟁점은 **역할 범위**:

| AI 역할 | 결과 |
|---|---|
| 예언자 (자세한 풀이 + 추가 대화) | 차별화 죽음, 비용·환각↑, 시장 따라가기 |
| 정리·연결 보조 (예언 X, 사전 내용 묶기만, 근거 노출) | 차별화 유지, 사용자 기대는 약함 |

- **고정 인식**: grounding(사전+질문에 답변 묶기)은 환각을 *줄이지* *제거하지 못한다*. 엮는 추론은 여전히 생성물. 따라서 AI를 붙이더라도 명분은 "정확해서"가 아니라 **"재미·편의 옵션이라서"**. 차별 축은 정확성이 아니라 **투명성**(근거 추적 가능)이다.
- **투명성 UI 원칙**: AI 문장이 어느 사전 의미에서 나왔는지 시각적으로 연결되어 보여야 함("이 문장 → 이 카드의 이 키워드"). 말이 아니라 화면으로 증명.

## 5. 의식 흐름 설계 (전통 근거 + 디지털 번안)

> 형식을 복제하지 않고 **효과를 번안**한다. 모드 분기로 가벼움/진지함 양쪽 수용.

| 단계 | 전통 | 디지털 번안 | 모드 분기 |
|---|---|---|---|
| ① 그라운딩 | 호흡·집중·질문 명료화. "신호가 선명할수록 메시지가 선명" | 진입 페이싱(서두르지 못하게) | 가벼운 모드는 축약 |
| ② 질문 | 의식 안에서 질문 → 몰입↑ | **질문 직접 적기**(=대가 의식, 기기 밖 미전송) + **도메인 칩 1개 선택**(결정론 매칭 입력) | 공통 |
| ③ 셔플 | 에너지 주입 + 무작위성이 핵심 | 직접 섞는 인터랙션. 점퍼 카드 연출은 v1.5 | 공통 |
| ④ 컷 | 왼손 3더미. **컷=패턴 고정(비가역)** | 되돌릴 수 없는 확정 연출. **각 카드 방향 비트가 이 시점에 숨은 채 고정** | 진지 모드 강조 |
| ⑤ **정/역 선택 ★시그니처** `[합의]` | **전통에 없음 — 사용자 오리지널.** 질문자가 뒤집기 전 정/역 선택 | 숨은 방향 × 선택 = 최종 방향 (아래 명세) | 진지 모드 핵심 |
| ⑥ 뽑기 | 비주류 손으로 끌리는 카드 | 스프레드 포지션 배치 | 공통 |
| ⑦ 리딩 | 키워드 노출 → 질문 연관만 강조 | 결정론 사전(도메인 필터) + (후속 옵션) AI 근거 연결 | 깊이 조절 |

**대가 의식** `[합의]`: **질문 직접 적기**로 번안 — 돈(복채) 대신 정성·시간이라는 대가. 공짜로 휙 받는 결과 vs 작은 대가를 치른 결과는 받아들이는 무게가 다르다.

**비가역성이 이 도구의 심장.** 시장 AI 타로는 다시 뽑기가 가벼움 → 우리는 컷·정/역 선택을 비가역으로 묶어 진지함을 만든다.

### 정/역 선택 메커니즘 명세 `[합의]`

물리 타로의 실제 역학을 그대로 옮긴다.

1. **컷 시점**: 각 카드의 방향(정/역)이 무작위로 결정되어 **숨은 채 고정**된다 (비가역).
2. **선택**: 질문자가 뒤집는 방향(정/역)을 고른다.
3. **최종 방향** = 숨은 방향 × 선택 — 선택이 숨은 상태를 그대로 보존하거나 전부 반전시킨다.

결과는 "운명(무작위)이면서 동시에 내 선택(책임)". 사용자 실경험의 "역방향을 골랐다면 결과는 정반대였을 것"이 정확히 이 구조. ⚠️ 단순 "방향 고르기" 버튼(선택=결과 직결)으로 구현하면 시그니처의 의미가 죽는다 — 반드시 2층 구조(숨은 상태 + 선택) 유지.

**커밋-리빌 검증** `[합의: MVP 포함]`: 선택 **전** 결과의 봉인 해시 표시 → 선택 **후** 검증 가능. "당신이 고른 뒤 결과를 바꾸지 않음을 증명" = 말 아닌 시스템 강제(brennhub 정신). 타 서비스 전무. 구현 비용 작음 — 도입 시점만 결정.

### 화면 와이어 개념 (MVP · 진지 모드) `[합의 2026-06-10]`

> 단일 페이지 상태 머신(라우트 이동 없는 단계 전환). 모바일 세로 우선. 입구→리딩 약 2분 — **느림이 곧 제품.**
> 화면 순서는 개념표(§5)의 ⑤⑥을 교차: **배치(S5)를 정/역 선택(S6)보다 앞에** — 눈앞에 놓인 카드에 운명을 거는 긴장감. (사용자 실경험도 "배열 후 뒤집기 전" 가능성 언급)

- **S0 입구** — 도구 소개 + 신뢰 한 줄("모든 해석은 공개된 사전에서 나옵니다") + [리딩 시작]. 보조 진입 [카드 사전 열람] `[합의]`: 22장 브라우즈 페이지 — 투명성 증명 + SEO 자산.
- **S1 그라운딩** — 어두운 전환, 느린 펄스 비주얼 + "마음속 질문을 하나로 좁혀보세요" 1줄. 약 6~8초 고정 페이싱, 스킵 없음. 진행바 금지(로딩처럼 보이면 실패) — 호흡 리듬 연출.
- **S2 질문** — 질문 직접 적기(textarea, 빈 질문 불가, "리딩 화면에 그대로 표시됩니다" 안내 = 정성 유도) + 도메인 칩 단일 선택(연애/일·직업/돈/관계/자기) + 마이크로카피 "질문은 이 기기 밖으로 나가지 않습니다".
- **S3 셔플** — 자동 애니메이션 금지, 드래그/스와이프로 직접 섞기. 최소 인터랙션(예: 3회) 후 [이제 됐어요] 버튼 등장 — 끝내는 시점도 사용자가 결정(전통: "됐다 싶을 때까지"). 엔트로피: crypto RNG + 인터랙션 좌표·타이밍 시드. 점퍼 카드 연출 자리(v1.5).
- **S4 컷** — 두 지점 탭/드래그로 3더미 분할 → 합칠 순서 탭 선택(순서가 실제 덱 순열을 결정 = 정직한 인터랙션). 합치는 순간 **카드 순서·방향 비트 확정(숨김)** + "이제 되돌릴 수 없습니다" 연출. 이후 뒤로가기 불가(수정 = 처음부터). 커밋-리빌 해시 표기 자리(도입 시 작은 봉인 문구).
- **S5 배치** — 덱 3회 탭 → 위에서 한 장씩 과거/현재/미래 자리로 뒷면 배치.
- **S6 정/역 선택 ★시그니처** — 뒷면 3장 위에 문구: "카드의 방향은 이미 정해져 있습니다. 당신의 선택이 그것을 그대로 두거나, 전부 뒤집습니다." [정방향으로 뒤집기] / [역방향으로 뒤집기]. **꾹 눌러(약 1.5초 홀드) 확정** — 의도적 마찰 = 선택의 무게. 더블 확인 모달 금지(긴장 희석).
- **S7 오픈** — 각 카드를 **사용자가 직접 탭해 뒤집음**, 과거→현재→미래 순서 강제. 뒤집힘 = 카드 비주얼 + 이름 + 방향 뱃지.
- **S8 리딩** — 상단: 적은 질문 원문(따옴표) + 도메인 뱃지 = 대가의 회수(질문이 리딩의 제목). 카드별 섹션: essence(항상) → 도메인 매칭 키워드 강조 + gloss(도메인 칩과 동일 색 연결 — §3 투명성 UI 원칙의 결정론 버전) → 비매칭 키워드는 접힘("전체 키워드" 토글: 숨기지 않되 강조만) → mute 케이스 정직 문구 → [원전 보기] 토글(Waite 원문). 하단: [이미지로 공유](질문 미포함 기본) / [새 리딩](처음부터 — **같은 리딩 다시 뽑기 없음**, 전통 "같은 질문 두 번 묻지 않기" 채택) / [사전 열람].

**상태 저장** — 마지막 리딩 localStorage(hydrate/persist/schemaVersion 패턴). 재방문 시 "지난 리딩 보기".
**모드 자리** — 가벼운 모드(데일리 1장)는 후속: S1·S3·S4 축약 + 선택 생략 버전. MVP 미포함.

## 6. 카드 사전 (DB) 설계 방향

- **MVP 범위** `[합의]`: **메이저 아르카나 22장** (22 × 정/역 = 44 항목 — 78장 대비 집필량 72%↓). 메이저 온리 리딩은 실존하는 타로 관행 — "굵직한 질문에 굵직한 카드" 명분 성립. 사용자 실경험의 두 카드(Strength·Death)도 메이저. 마이너 56장은 v2 콘텐츠 확장.
- **구조**: 22장 × {정방향, 역방향} × {다중 키워드}. 키워드마다 도메인 태그. 도메인 5개 안: 연애 / 일·직업 / 돈 / 관계 / 자기 `[제안]`.
- **질문 처리** `[합의]`: 자유 질문 **직접 적기**(의식·대가·리딩 화면 표시용 — 텍스트 분석 안 함) + **도메인 칩 1개 선택**(결정론 키워드 필터). 가짜 NLP 금지 — 질문을 이해하는 척하지 않는다. 사람의 섬세한 선별에 못 미치는 한계는 인지하되, 정직을 택한다.
- **정직성 장치** `[합의]`: 선택 도메인에 닿는 키워드가 없는 카드는 억지 해석 금지 → "이 카드는 [도메인]에 직접 닿는 키워드가 없어요. 전체 의미로 보세요." (mute card 정신의 MVP 번안)
- **출처 전략** `[합의]`: 1910 퍼블릭 도메인 원전(Waite, *The Pictorial Key to the Tarot*) 기반 + 사용자 큐레이션 → **"출처 있는 사전"** 신뢰 문구 가능 + 저작권 클린. 집필 파이프라인: 자문 Claude 초안 → 사용자 편집장 검수. (런타임 AI 아님 — 결정론 원칙과 무관)
- **출처 3층 구분 + 원전 검증** `[합의 2026-06-10]`: ① 원전 인용층 — 1910 원문 §3.3 전문, 독립 전사본 2종 대조 완료 ② 점성·원소층 — 골든던 전통(원전 책에 없음) ③ 해석층 — 현대 통용 합의 기반 큐레이션(단일 공식 출처 부재 — 편집장 승인 + 외부 AI 교차 검증으로 담보). 상세: `tarot-dictionary.md` 헤더.
- **프라이버시** `[합의]`: 적은 질문은 기기 밖 미전송(localStorage만). 리딩 결과 서버 저장 없음(MVP). 공유는 이미지 카드만.
- **스키마**: 정적 JSON 번들(클라이언트). brennhub 패턴(localStorage hydrate/persist/schemaVersion) 재사용.
- **상징 메타**: 카드별 상징·원소·수비학(흥미 콘텐츠 + 깊이). RWS 기반(1909, 퍼블릭 도메인).

### 6.1 스키마 `[합의 2026-06-10]`

- **카드 의미는 포지션 비의존** — 포지션(과거/현재/미래)은 표시 라벨일 뿐, 사전 항목은 카드×방향 단위. (고교 일화 방식과 동일: 해설집은 하나, 자리는 읽는 틀)
- 모든 텍스트 필드는 `{ ko, en? }` 형태 — §8 "카드 본문 i18n 범위" 결정과 무관하게 스키마 변경 없이 en 추가 가능.
- 기획 단계에선 산문(본 문서), 구현 착수 시 정적 JSON으로 이관.

```ts
type Domain = "love" | "work" | "money" | "relation" | "self";
// 칩 라벨: 연애 / 일·직업 / 돈 / 관계 / 자기

interface TarotCard {
  id: number;                 // 0~21 (메이저 번호)
  slug: string;               // "strength", "death" ...
  name: { ko: string; en: string };
  meta: {                     // 흥미·깊이 콘텐츠 (리딩 보조)
    numerology: number;
    element?: { ko: string };
    astrology?: { ko: string };
    symbols: { ko: string }[];   // 그림 속 상징 1줄 노트
  };
  upright: OrientationEntry;
  reversed: OrientationEntry;
}

interface OrientationEntry {
  essence: { ko: string; en?: string };   // 2~3문장 핵심 의미
  waite: string;                          // 1910 원전 점술 의미 원문(영문) — "출처 있는 사전"의 실체
  keywords: {
    word: { ko: string; en?: string };
    domains: Domain[];                    // 도메인 칩 매칭 대상
    gloss: { ko: string; en?: string };   // 1문장 건조 풀이 (텔레그램식)
  }[];                                    // 방향당 4~6개 권장
}
```

- **리딩 로직**: 선택 도메인 ∈ `keyword.domains` → 해당 키워드 강조 + gloss 노출. essence는 항상 노출. 매칭 0개면 mute 문구 + essence만 (§6 정직성 장치).

#### VIII. 힘 (Strength)

**meta** — 수비학 8 · 원소 불 · 점성 사자자리 · 상징: 사자의 입을 다물게 하는 여인(누르는 힘이 아닌 다스림), 머리 위 ∞(마르지 않는 내면의 힘), 흰 옷(순수한 의지), 꽃 화환(길들여진 본능)

#### XIII. 죽음 (Death)

**meta** — 수비학 13 · 원소 물 · 점성 전갈자리 · 상징: 흰 장미의 검은 깃발(끝 속의 생명), 두 탑 사이로 뜨는 해(끝 너머의 새벽), 갑옷의 해골 기사(피할 수 없음), 쓰러진 왕(지위도 예외 없음)

| 키워드 | 도메인 | gloss |
|---|---|---|
| 끝맺음 | 연애, 관계 | 이 관계의 한 형태가 수명을 다했다. |
| 전환 | 일·직업, 자기 | 이전 방식과의 단절. 새 국면의 입구. |
| 손절 | 돈 | 미련 없이 정리해야 더 큰 손실을 막는다. |

| 매듭 | 일·직업 | 끌고 온 일을 끝내는 시점. |

**역방향** — essence: "끝내야 할 것을 끝내지 못하는 정체. 변화를 거부하는 동안 에너지가 새고, 멈춰 있음이 가짜 안전감을 준다."
원전(Waite): "Inertia, sleep, lethargy, petrifaction; hope destroyed."

| 키워드 | 도메인 | gloss |
|---|---|---|
| 미련 | 연애, 관계 | 끝난 것을 붙잡고 있다. |
| 정체 | 일·직업, 자기 | 끝나지도 시작되지도 않는 상태. |

| 변화 거부 | 자기 | 멈춰 있음이 주는 가짜 안도. |
| 소진 | 자기, 일·직업 | 버티기만 하다 에너지가 닳는다. |

### 6.3 검증 시뮬레이션 — 실경험 재현

조건: 도메인 칩 **연애**, 카드 **힘 정방향 + 죽음 정방향** (고교 일화와 동일 입력).

- 힘 정방향 → 연애 매칭 키워드 **부드러운 끈기** 강조: "밀어붙임이 아니라 다정한 꾸준함이 상대를 움직인다." → 당시 "여자애의 마음" 해석과 일치.
- 죽음 정방향 → 연애 매칭 키워드 **끝맺음** 강조: "이 관계의 한 형태가 수명을 다했다." → 당시 "남자애의 마음" 해석과 일치.

→ **건조한 사전 + 도메인 필터만으로 실제 리딩이 재현된다.** 스키마 1차 검증 통과. (mute 케이스 예: 어떤 카드 방향에 [돈] 태그 키워드가 없으면 → mute 문구 + essence만 노출)

## 8. 열린 결정사항 (다음 라운드)

> 아래 4건은 편집장 직접 검증이 어려운 영역 — 자문 추천 일괄 수용(2026-06-10). dev 실물 확인 후 피드백으로 재조정 가능.

- **카드 본문 i18n 범위** `[합의 2026-06-10]`: **UI 크롬(버튼·라벨·안내·의식 문구)은 ko/en 필수 이행, 사전 본문(essence·키워드·gloss)은 ko 전용 출시.** en 본문은 백로그 — 본문 톤(건조한 텔레그램체)은 번역이 아니라 재집필이고, 영어권 타로 콘텐츠는 포화 시장이라 SEO 효익도 없음. 스키마 `{ko, en?}`로 미래 호환은 이미 확보. en locale에서 카드 이름은 영문 데이터 사용, 본문은 ko 노출. 의도된 예외로 백로그 명기 (선례: saju-naming 한국어 우선). 도메인 5종 칩 라벨은 UI 크롬으로 ko/en, enum 값은 내부 고정.
- **비주얼** `[합의 2026-06-10]`: **선출시 = CSS 타이포 카드** — 번호(로마자)·이름·원소/점성 글리프·다크 테두리를 카드 데이터로 렌더링 + 뒷면 패턴 1종(S5·S6의 주인공). 제작 병목 0, 의식 무드와 통일, 다크모드 자연 대응. 픽셀아트 덱은 v1.x 교체 트랙(편집장 직접 제작 시). RWS 1909 도판 스캔(퍼블릭 도메인 진입)은 카드 앞면 비채택 — 전 경쟁 앱과 동일 비주얼 = 차별 죽음. 사전 열람의 '원전 도판 참고' 용도만 백로그 옵션.
- **커밋-리빌 해시 도입 시점** `[합의 2026-06-10]`: **MVP 포함.** 컷 확정 시 (카드 순서 + 방향 비트 + nonce)의 SHA-256을 S4에 봉인 문구로 표시 → 리딩 화면의 접힌 '검증' 토글에서 원본+nonce 공개. Web Crypto, client-side, 외부 의존 0. '말 아닌 시스템 강제'의 제품화이자 S6 선택의 무게에 대한 기술 증명. 단 일정 압박 시 1순위 컷 후보로 명기(절제 장치).
- **AI 역할 범위** `[열림]`: §3 표 기준. 기본 구축 완료 후 재론.
- **정식 명명** `[합의 2026-06-10]`: 한국어 이름 **"타로 테이블"** / 슬러그 **`tarot`**. 슬러그 = 최단 SEO 경로 + brennhub 단순성. '테이블'은 셔플·컷·배치·오픈이 일어나는 의식의 공간 — 화면을 테이블화하는 제품 본질과 정합. 대안(미채택): '타로 리딩'(안전·무미) / '타로 의식'(차별 직격·무게감). "AI" 미노출 자동 충족(런타임 AI 없음).
