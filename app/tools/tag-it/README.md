# 태그잇 (Tag-it) · `tag-it`

오피스 문서(Word·Excel·PowerPoint)에서 핵심 키워드를 뽑아 칩으로 다듬고, 문서 속성(keywords)에 기록해 다시 내려받는 도구.

## 한 줄

.docx / .xlsx / .pptx 업로드 → **브라우저 안에서** 본문을 읽어 키워드 후보를 칩으로 추출 → 사용자가 클릭·입력으로 다듬은 태그를 `docProps/core.xml`의 `<cp:keywords>`에 기록 → 다운로드. **서버 전송·AI 없음.**

OOXML 세 포맷은 `docProps/core.xml`의 keywords 위치가 동일 → 읽기·쓰기·재포장은 공유, 포맷별로 다른 건 본문 추출뿐.

## 아키텍처

- **클라이언트 전용**: 파일이 서버로 가지 않음. 파싱·추출·재포장 전부 브라우저. D1 미사용.
- **결정론적 계산**: 추출은 전부 사전 + 통계. 환각 0, 동일 입력 = 동일 결과 (saju-naming 철학).
- **서버 무상태**: 새로고침 = 초기화. localStorage엔 고급 옵션값만 저장 (파일 내용 X).
- 스택: Next.js 16 + Cloudflare Workers(정적 서빙) + Tailwind v4. 신규 인프라 0.

## 모듈 (`lib/tag-it/`)

| 파일 | 역할 |
|---|---|
| `limits.ts` | 한계값 단일 소스 (파일 5개 / 8MB / 15MB / 노출 30 / 태그 50개·30자) |
| `stopwords.ko.ts` | 한국어 기능어 3층 (조사 / 용언 어미 / 의존명사·부사) — 단어만 추가 가능 |
| `stopwords.en.ts` | 영어 표준 불용어 ~120 |
| `tokenize.ts` | 토큰 분리 + 조사 longest-match 제거 (**과도제거 가드**: 남는 어간 2음절 미만이면 원형 보존) |
| `extract.ts` | 점수화(빈도 + 위치 가산점) + **명사 확률 모델**(사전 보호 + 어미·신호 가중 차감) + 중복 병합 |
| `weights.ts` | 명사 확률 모델 가중치 **부록** — 어미 가중치 = 사전 명사 충돌 역수. 코드 수정 없이 숫자만 조정 |
| `noun-dict.ts` | NNG 명사 사전 lazy-load 싱글턴 (정적 gzip → fflate → Set). **fetch 실패 fallback** |
| `chips.ts` | 후보 → 칩 변환 / 재추출 시 사용자 칩(채택·보호·수동) 보존 / 최종 태그 산출 |
| `office.ts` | fflate zip 해제 + 포맷 디스패치(`detectFormat`) → 본문 추출(docx 본문/표, xlsx sharedStrings+inline, pptx 슬라이드 자연정렬) / core.xml keywords 읽기·쓰기·재포장(포맷 공유) / 다운로드 MIME |
| `storage.ts` | 고급 옵션 localStorage 영속 (값만) |
| `types.ts` | 칩 상태·파일·옵션 타입 |

## 재포장 무결성 (제일 깨지기 쉬운 지점, 세 포맷 공통)

- unzip이 돌려준 **각 엔트리의 콘텐츠 바이트를 그대로** 다시 zip에 넘기고, `docProps/core.xml` **한 엔트리만** 교체.
- 본문 XML(`word/document.xml` · `xl/*` · `ppt/slides/*`)은 **절대 건드리지 않음** → "문서 손상" 경고 위험 최소화.
- core.xml의 `<cp:keywords>`만 교체/삽입. core.xml이 아예 없는 경우(드묾)에만 생성 + `[Content_Types].xml`·`_rels/.rels` 등록.
- 재압축 자체는 정상 (Word도 저장 시 재압축). 콘텐츠·경로·구조가 보존되면 호환 뷰어가 손상 없이 연다.

## 키워드 추출

1. **토큰화 + 조사 제거**: 공백·구두점 분리. 한국어는 longest-match로 조사 제거(과도제거 가드 포함). 혼합 토큰("30분"·"AI보고서")은 보존.
2. **명사 어간 복원 + 형태 노이즈 제거**: "분석했다→분석" 류 명사+하다/되다 복원. 기호·숫자단편·세는단위·URL·불용어는 토큰 형태 기준 제외(강도별 게이트).
3. **명사 확률 모델**: 빈도 + 제목 가산점으로 점수화. 명사 여부는 **0~100 확률**로 — 아래 참고. 중복은 정규화 키로 병합.
4. **언어**: 한국어·영어 전용 사전. 그 외 언어는 fallback(공백·구두점 분리 + 빈도).

### 명사 확률 모델 (0.8.0)

"세상은 흑백이 아니다." 용언 어미를 제거/보존 이분법으로 자르지 않고, **NNG 명사 사전을 보호
신호**로 써 확률을 매긴다. (`extract.ts` §nounProbability / `weights.ts` / `noun-dict.ts`)

- **사전 O**(mecab-ko-dic NNG 216,932개): 100 시작, 하한 **85**(차감 ≤15). 동음이의(하지=夏至)도
  보호하되 호버 %로 불확실성 노출.
- **사전 X**: 50 시작, 어미·신호1 가중 차감(하한 0).
- **가중치 = 사전 명사 충돌 역수** — 충돌 적은 어미(했다/입니다/적인…)는 강한 차감, 명사 말음
  (지/음/어/고/면…)은 0(차감 금지). `weights.ts`에서 코드 수정 없이 조정.
- **강도 = 채택 임계**(3:20 / 4:30 / 5:40). 강도 1~2는 임계 0(관대). 임계 40 < 비사전 명사 50 →
  비사전 명사는 전 강도 보존, 노이즈(≤25)만 단계적 제거.
- **사전**은 정적 자산 1회 lazy-load(0.75MB gzip). **fetch 실패 시 fallback**(50% 경로 = 0.7.x
  동작, 안 깨짐). 문서는 서버로 전송되지 않음. 출처·라이선스: `data/NOTICE`(Apache-2.0).
- **UI**: 후보 칩 hover 시에만 신뢰도 % 노출(70↑ 초록 / ↓ 호박색).

## 칩 캔버스

- 상태: 후보(희미) / 채택(강조) / 보호(주황, 필터 면제) / (숨김은 2차).
- 조작: 본체 클릭 = 채택 토글 / 핀 = 보호 / × = 삭제 / 직접 입력 = 채택 칩 추가.
- 모드: 자동(추출 칩 채움) / 수동(빈 캔버스). 모드 전환 시 기존 칩 보존.
- 점진 노출: 상위 30개 먼저 + "더보기".

## 복수 파일 · 다운로드

- 공통 트레이(전 파일 일괄 적용) + 파일별 카드(개별 칩 + 상태 배지).
- 개별 다운로드(원본명) / 복수 시 전체 .zip 일괄.

자세한 배경·결정 근거는 기획서(`tag-it-plan.md`)와 루트 `BRENNHUB.md` 참고.
