단계별(P3→P1→P2→P4) 구현 + 단계마다 빌드 통과.

- **P3 세그먼트 컨트롤**: 필터 강도 슬라이더 → 가로 5칸 세그먼트(`segmented.tsx`, 강도·모드 공용).
  칸 통째 클릭(Fitts) · radiogroup + ←→ 키보드 · 현재 단계명 + 한 줄 힌트.
- **P1 시각 위계 3층**: ① 입력(업로드) → ② 추출 설정(접힘, 강도·범위·최소빈도) → ③ 주 작업공간(결과).
  모드(자동/수동)는 추출 설정에서 빼 **작업공간 헤더(칩 바로 위)에 상시 노출** — 근본 갈림길이라 발견성 우선.
  작업공간 상시 노출 + 빈 상태 안내.
- **P2 후보 밀도**: 칩면 빈도 숫자 제거(hover 툴팁으로만) · 후보 구역에 **칩 검색**(실시간 부분일치) ·
  기본 노출 30 → 20 · 더보기/검색 병행.
- **P4 마감**: 채택=채움 / 후보=외곽선(투명) 색+형태 이중 구분(색맹 안전) · 옵션·모드 변경 시
  "갱신됨" 마이크로 펄스(절제) · 빈 상태/검색없음 문구.

## 0.4.0 — 추출 품질 2차 보강 + 필터 강도 슬라이더 (2026-05-29)

**A. 잔여 노이즈 (형태 기반, 문서 무관)**
- 조사 제거를 영문/숫자 어근에도 적용(`tokenize.ts`): `gd에→gd`, `70과→70`, `preload만으로→preload`.
  어간 유효성 = 한글 ≥2음절 또는 영문·숫자 포함. 스택 조사 반복 제거. 한글 1음절 어근(회의/수로)은 가드로 보존.
- 활용형 어미 보강(`stopwords.ko.ts`): 되는/하는/지는/되며/된 등(관형형·피동). `~ㄹ` 관형형(건드릴)은
  명사 충돌(드릴·소설·수단)로 결정론 안전 제거 불가 → 추가 안 함(명사 보존 우선).

**B. "명사 위주" 토글 → 5단계 "필터 강도" 슬라이더**
- 1 관대 ↔ 5 엄격. 강도→필터 프로파일(`extract.ts` PROFILES): 강도별로 숫자영문단편/조사·활용형·세는단위/
  약어감점/최소빈도 floor/약어제외를 점증 적용. 기호·비문자·기능어 제거는 전 강도 항상.
- 기본값 3(균형, 기존 동작 수준). 슬라이더 + 현재 단계명 + 한 줄 힌트(무엇을 거르는지) i18n(ko/en).
- removeJosa 토글을 강도에 흡수(별도 토글 제거). minFreq 스텝퍼는 유지(강도가 내부 floor 추가).
- localStorage 옵션 스키마 v1→v2(구버전 값 무시, 기본 강도 3로 1회 리셋).

## 0.3.0 — 추출 품질 보강 (2026-05-29)

dev 실사용 결과 "명사 위주 ON"인데도 활용형·숫자결합·기호파편이 후보로 노출되던 문제.
문서 종류를 가리지 않는 **형태 기반 보편 규칙**으로 노이즈 제거 (코드/산문 분기 없음).

- **활용형 어미 확장**(`stopwords.ko.ts` `KO_VERB_ENDINGS`): 이라/으면/하면/해야/하지/했던/는데/라는 등
  2글자+ 위주 추가(명사 위주 ON일 때 제외). 1글자는 충돌 적은 `던`만 — 이미지·편지·측면·신라 등 명사 보호.
- **형태 노이즈 필터 3종**(`extract.ts`, 명사위주 무관 항상 적용):
  - 기호·비문자 토큰(i→, s×3s) — 유니코드 글자 기준이라 fallback 언어 안전.
  - 숫자+영문 단편(5px, 0s, F1) — 한글 포함 토큰(AI보고서·Q3실적)은 면제.
  - 숫자+세는단위(1줄/2가지/3번/1개) — 시간·물리 단위(30분·2개월)는 보존.
- **약어 약한 감점**(`extract.ts`): gd/dps/int 등은 제외하지 않고 점수 -1 — 빈출 약어(MRI 등)는 상위 유지.
- 어미가 붙은 활용형은 **어간 복원이 아니라 제외**(결정론·안전 우선). "후보에서 골라담기" 구조라 충분.

## 0.1.0 — MVP 1차 (2026-05-29)

신규 도구 추가. .docx 키워드 칩 추출·편집·기록 (클라이언트 전용, AI 미사용).

**.docx 처리 (`lib/tag-it/docx.ts`)**
- fflate 기반 zip 해제 → 본문(`word/document.xml`) + 표 텍스트 추출.
- core.xml `<cp:keywords>` 읽기·쓰기. **core.xml 한 엔트리만 교체, 나머지 바이트 보존** → 본문 무결성 유지 (손상 경고 방지).
- 여러 .docx 일괄 .zip 묶음(level 0, 재압축 회피).

**추출 엔진 (`lib/tag-it/`)**
- 토큰화 + 조사 longest-match 제거. **과도제거 가드**: 떼고 남는 어간이 2음절 미만이면 원형 보존("회의→회" 방지).
- 점수화(빈도 + 제목 영역 가산점) + 명사 위주 네거티브 필터(용언 어미 제외) + 중복 병합.
- 한국어·영어 전용 불용어 3층 사전(코드와 분리, 단어만 추가 가능). 그 외 언어 fallback.

**UI**
- 칩 캔버스: 후보/채택/보호/숨김 상태 모델, 본체클릭=채택·핀=보호·×=삭제·직접입력.
- 자동/수동 모드 토글 (모드 전환 시 사용자 칩 보존).
- 고급 옵션(조사 제거·명사 위주·읽기 범위·최소 빈도) + localStorage 영속.
- 점진 노출(상위 30 + 더보기). 업로드 한계 검증(파일 5개/8MB/15MB, 어느 한계인지 구분 경고).
- 복수 파일: 공통 트레이(일괄 적용) + 파일별 카드(상태 배지) + 개별/일괄 다운로드.

**brennhub 통합**
- registry(`status:"live"`, `hasPage:true`) + page + 문서 3종.
- i18n `tagIt` namespace (ko/en) + tools + feedback(`toolTagIt`) 6곳 배선.
- 의존성 `fflate` 추가 (브라우저 zip — 자체 구현이 유지보수성 해침, 기획서 §3.1 명시 예외).
