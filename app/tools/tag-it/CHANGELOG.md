## 0.8.0 — 명사 확률 모델 + NNG 사전 보호 (2026-05-30)

"세상은 흑백이 아니다" — 용언 어미를 **제거/보존 이분법**으로 자르던 것을, NNG 명사 사전을
**보호 신호**로 쓰는 **확률 모델(0~100)**로 교체. 사전은 규칙을 버릴 대상이 아니라 규칙을
검증·보호하는 토대(가중치 = 사전 명사 충돌 역수). 여전히 AI 0·결정론·클라이언트 전용.

- **명사 사전(`noun-dict.ts`)**: mecab-ko-dic 2.1.1 NNG 일반명사 216,932개(한글 음절만) →
  gzip 0.75MB 정적 자산(`public/tag-it/noun-dict.txt.gz`). 클라이언트 1회 lazy-load(fflate
  gunzip) → Set. **fetch/디코드 실패해도 안 깨짐**(fallback: 사전 없이 50% 경로 = 0.7.x 동작).
  Apache-2.0 — `data/NOTICE`·`data/LICENSE-mecab-ko-dic` 동봉. 생성: `scripts/gen-noun-dict.mjs`.
- **확률 모델(`extract.ts` §nounProbability)**:
  - 사전 O: 100 시작 · 하한 **85**(차감 ≤15, 추락 봉쇄) — 동음이의(하지=夏至 등)도 보호하되
    호버 %로 불확실성 정직하게 노출.
  - 사전 X: 50 시작 · 강차감(하한 0). 어미(longest-match)+신호1(용언 패러다임) 가중 차감.
  - **강도 = 채택 임계**(THRESHOLD 3:20/4:30/5:40). 강도 1~2는 0(관대, 기존 동작).
  - 기존 binary `isVerbLike`/신호1 제거 로직을 모두 가중 차감으로 흡수.
- **가중치 부록(`weights.ts`)**: `{어미, 가중치, 충돌}` 표 — **코드 수정 없이 숫자만 조정**.
  충돌 실측(NNG 216,932): 0충돌 종결/연결/인용 어미(했다/입니다/하는/적인…)=45·35, 1글자
  명사 말음(지=3365/음=1103/어=1082/고=684/면=524…)=0(차감 금지, 가게/녹음/창고 보호).
- **UI(`chip.tsx`)**: 후보 칩 **hover 시에만** 명사 신뢰도 % 노출(평소 비노출) — 70%↑ 초록(사전
  보호)·↓ 호박색(규칙 판정). 툴팁에도 `신뢰도 N% · M회 등장`. i18n ko/en.

검증(시나리오): 법인/투자/고객/서비스/계약/신뢰/분석/평가/연구=100% · **투자금 50% 보존**(50점
경계 해소: 비사전 명사 50 vs 노이즈 ≤25 간극 + 임계 40<50) · 객관적인/사랑스러운/혁신적인/
먹었다 제거 · **하지=85%**(사전 보호) · 신뢰했→신뢰 흡수 · 명사 보존 30/30 · 강도 단조성

## 0.6.2 — 추출 정밀화: 명사 어간 추출 + 강도·빈도 분리 (2026-05-29)

실파일 계측으로 3개 독립 원인 확정 후 우선순위 #2→#3→#1 수정.

- **#2 명사 어간 추출(핵심)**: 용언("도출하였다")을 통째 버리던 것을 → 접미 용언(하/되/시키 계열)을
  떼어 명사 어간 복원("도출"/"분석"/"평가"). `KO_NOUN_VERB_SUFFIXES` + `extractNounStem`.
  - 과분리 3중 가드: 남는 어간 한글 ≥2일 때만 분리(위하여→위 미분리), 접미는 하/되/시키 한정
    (먹었다 등 일반 동사 미분리), 1음절-stem 명사(북한/포함/결함/이해) 보존.
  - 조사 제거보다 **먼저** 실행 — "포함하는"의 "는"이 조사로 떨어져 "포함하"가 되기 전에 "포함" 복원.
  - 효과: 직전 문단-join 수정으로 역으로 사라졌던 명사 부활 + 용언에 갇힌 명사 대량 복원(recall↑).
- **#3 강도-빈도 축 분리**: `FilterProfile`에서 `minFreqFloor` 제거. 강도 4·5의 floor=2가 1회 등장
  명사를 전멸시키던 문제 해소 → 강도는 "명사 판별 엄격도"만, 빈도 컷은 사용자 `minFreq` 옵션만.
  강도 5에서도 1회 명사 생존. (단조성은 필터 strictness로 유지)
- **#1 사전 안전 보강**: 복합조사 `으로부터`/`로부터` `KO_JOSA` 추가, 단독 의존명사 `번째` stopword 추가.

## 0.6.1 — 추출 버그 수정: 문단 단위 join (pptx/xlsx 토막 복원) (2026-05-29)

**증상**: pptx(및 리치텍스트 xlsx)에서 강도 5인데도 조사·어미 노이즈(에서/로부터/하지/
도출되/돈을/객관적인…)가 대량 후보로. **원인**: `extractByTag`가 런(`<a:t>`/`<w:t>`/`<t>`)을
공백으로 join → PowerPoint가 단어 중간을 여러 런으로 쪼개면 단어가 토막나고, 분리된
조사/어미 조각이 "어근에 붙은 조사/어미"를 가정한 필터를 빠져나감. (엔진은 포맷 무관 동일,
본문 추출의 토큰 경계 손상이 원인 — 코드 경로 분기 아님.)

**수정** (`office.ts`, 방향 1 = 문단 단위 join):
- `extractByTag` → `extractParagraphs(xml, paraTag, runTag)`. 문단/셀(`<w:p>`/`<a:p>`/`<si>`)
  **안의 런은 공백 없이("") 연결**(단어 복원), **문단·셀 사이만 공백**.
- 근거: OOXML에서 단어 띄어쓰기는 항상 런 내용 안의 문자(xml:space="preserve" 포함) →
  ""로 이어도 진짜 공백 보존 + 토막만 복원.
- 줄바꿈·탭(`<w:br/>`/`<w:tab/>`/`<a:br/>`)은 run이 아니라 공백 런으로 치환 → 단어 병합 방지.
- run 태그만 추출 유지(`<w:instrText>` 등 비가시 필드 제외) → **docx 회귀 0**(단일런 불변,
  여러 런 문단은 복원되어 개선).
- 3포맷 동일 적용 → 추출 결과 포맷 무관 정규화("문서는 문서다").

**검증**: 동일 의미 문장을 docx/pptx(런 토막)/xlsx(리치텍스트 셀)로 만들어 추출 원문·토큰이
**완전 동일**함을 스모크로 확인. 진단 노이즈(에서/로부터/도출하/였다) 소멸, 명사(매출/고객/
분석/보고서/도출) 보존, 줄바꿈·탭 병합 없음. build + lint 통과.
