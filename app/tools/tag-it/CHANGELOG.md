# 태그잇 (Tag-it) CHANGELOG

## 0.1.0 — MVP 1차 (2026-05-29)

신규 도구 추가. .docx 키워드 칩 추출·편집·기록 (클라이언트 전용, AI 미사용).

**.docx 처리 (`lib/tag-it/docx.ts`)**
- fflate 기반 zip 해제 → 본문(`word/document.xml`) + 표 텍스트 추출.
- core.xml `<cp:keywords>` 읽기·쓰기. **core.xml 한 엔트리만 교체, 나머지 바이트 보존** → 본문 무결성 유지 (손상 경고 방지).
- 여러 .docx 일괄 .zip 묶음(level 0, 재압축 회피).

**추출 엔진 (`lib/tag-it/`)**
- 토큰화 + 조사 longest-match 제거. **과도제거 가드**: 떼고 남는 어간이 2음절 미만이면 원형 보존("회의→회" 방지).
- 점수화(빈도 + 제목 영역 가산점) + 명사 위주 네거티브 필터(용언 어미 제외) + 중복 병합.
- 한국어·영어 전용 불용어 3층 사전(코드와 분리, 단어만 추가 가능). 그 외 언어 fallback.

**UI**
- 칩 캔버스: 후보/채택/보호/숨김 상태 모델, 본체클릭=채택·핀=보호·×=삭제·직접입력.
- 자동/수동 모드 토글 (모드 전환 시 사용자 칩 보존).
- 고급 옵션(조사 제거·명사 위주·읽기 범위·최소 빈도) + localStorage 영속.
- 점진 노출(상위 30 + 더보기). 업로드 한계 검증(파일 5개/8MB/15MB, 어느 한계인지 구분 경고).
- 복수 파일: 공통 트레이(일괄 적용) + 파일별 카드(상태 배지) + 개별/일괄 다운로드.

**brennhub 통합**
- registry(`status:"live"`, `hasPage:true`) + page + 문서 3종.
- i18n `tagIt` namespace (ko/en) + tools + feedback(`toolTagIt`) 6곳 배선.
- 의존성 `fflate` 추가 (브라우저 zip — 자체 구현이 유지보수성 해침, 기획서 §3.1 명시 예외).
