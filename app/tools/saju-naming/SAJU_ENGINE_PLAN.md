# 사주 명리 엔진 격상 — 마스터 플랜

> **상태**: 초안 v0.1 (정찰 정정 2건 반영, 검토 대기)
> **연관 도구**: `app/tools/saju-naming`
> **성격**: 본 문서는 명리 엔진 격상 프로젝트의 single source of truth. thread/인원이 바뀌어도 이 문서 하나로 전체 맥락이 인수인계된다. 결정과 보류 사유를 여기에 박아 6개월 뒤에도 흔들리지 않게 한다.

## 0. 개요

**목표**: saju-naming을 작명 수준 명리 엔진으로 격상. 사주 명식 도출 → 분석 → 작명·택일의 전체 파이프라인을 정밀화한다.

**범위**: 작명(기존) + 택일(신규) + 명식 정밀화 + 풀이 기반 강화.

**핵심 성격**: 직관감이 아닌 **결정론적 명리 계산**. 같은 입력은 항상 같은 출력. 명리 근거를 담담히 제시하고, "우주가 어쩌니" 류의 검증 불가 단언은 배제한다. 이 결정론성이 LLM 작명 대비 본 서비스의 차별점이다.

## 1. 아키텍처 결정 (확정 — 변경 시 사유 필수)

1. **결정론 우선**: 모든 항목(용신·충·합·길흉)은 고정된 규칙/데이터로. 선택지가 갈리는 항목은 brennhub이 채택 기준을 **명시적으로 고정**한다.
2. **코어 공유 구조**: 명식 도출(A)·분석(B)의 공통 코어 모듈. 작명·택일이 각각 호출. 로직 중복 없음(유지보수성).
3. **음령오행 학파**: 기존 **통용(신경준 운해본) 유지**. 해례본을 **학파 옵션으로 추가** (`sound-ohaeng.ts` plug-in 구조 활용). 기존을 손대지 않고 확장.
4. **데이터 환각 절대 금지**: 절기 시각·절입·길흉화 등은 **검증된 출처만** 사용. 추측값 사용 시 전체 결과 신뢰도 붕괴.
5. **기술 제약** (brennhub 금기 준수):
   - `export const runtime = 'edge'` **금지** (OpenNext 미지원, Turbopack silent broken bundle — CHANGELOG 0.5.x~0.6.2)
   - ORM(Drizzle/Kysely) 미사용 — D1 직접 쿼리 패턴 유지
   - Swiss Ephemeris Edge 포팅 대신 **정적 절기 JSON/D1** 사용
   - 진태양시는 **균시차 포함** (외부 기획의 누락분 보완)
6. **기존 자산 보존**: `won_stroke`(원획법), 한자 DB 9,460자, 음령오행 통용, 81수리, 음양오행 필터, 부적합 한자 가드(39-C) — 재사용. 새로 만들지 않음.

## 5. 영역 D — 택일 엔진 (신규)

**파이프라인**:
1. 대상 기간 각 날짜 명식(일진·시진) 도출 → A 엔진 재사용
2. **하드 배제**: 사용자 일진/월진 vs 대상일 충·형·파·원진 → 블랙리스트
3. **가산**: 일합/육합/방합(상생 실현=용신 차원에서 최대) + 용신 일치 + 길신(천을귀인 등)
4. **시진 반영**: 대상일이 아침 달일 시주도 사용시 충 체크
5. **길수 산정**: 합격일 내 시진별 점수 → 최상 2시간 권고
6. **출력**: 날짜별 점수 + **근거 명시** + 추천 길수. 픽튠 등 명리 근거만.

**구조**: 작명과 코어(A·B) 공유. 도구 노출은 별도 슬러그(`/tools/saju-taekil`) — 또는 saju-naming 내 탭. (결정 필요)
